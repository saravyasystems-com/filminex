# Security Policy

## Reporting

Do not disclose security vulnerabilities in public issues. Report them privately to the Saravya Systems repository administrators with reproduction steps, impact, and any suggested mitigation.

## Baseline rules

- Never commit credentials, API keys, tokens, or private media assets.
- Use least-privilege access for external AI and production tools.
- Separate tenant data and enforce authorization at every data boundary.
- Treat uploaded scripts, images, audio, and video as untrusted input.
- Record security-relevant mutations in an auditable event log.
- Define retention and deletion rules before storing production assets.

Supported versions and a dedicated security contact will be published before the first public release.
