## Lifecycle

`Proposed → Accepted → Frozen → Superseded`

Frozen ADRs are not edited to change their decisions. A future architectural change creates a new ADR naming the decision it supersedes. Implementation details and unresolved legal, governance, provider, version, and schema choices remain outside the frozen baseline.
